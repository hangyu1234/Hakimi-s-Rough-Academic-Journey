## 一、什么是markdown
这是一种纯文本格式，用于写结构化文档，基于email和Usenet[^1]的格式规范

## 二、谁创造了Markdown
它由 John Gruber 与 Aaron Swartz 于 2004 年开发

### 2. 段落与换行
**==注意：这里有一些问题，我在下载了Markdown PDF后，这里的部分知识可能有些偏差==**

### 3. 文本样式
1. **粗体** 或者 __粗体__ （\*\*粗体\*\* or \_\_粗体\_\_）
2. *斜体* 或者 _斜体_（\*斜体\* or \_斜体\_）
3. ***粗斜体*** （\*\*\*粗斜体\*\*\*）
4. ~~删除线~~ （\~\~删除线\~\~）
5. `行内代码` （\`行内代码\`）  
   ```
   多行代码
   ```  
   （\`\`\`多行代码\`\`\`）
6. ==高亮文本== （\=\=高亮文本\=\=）（部分平台支持）
7. <u>下划线</u> （\<u\>下划线\</u\>）
8. 字体、字号、颜色 （\<font face="字体" color="颜色" size="字体大小（数字）"\> 待添加文本 \</font\>）
    <font face="宋体" color="red" size="3">你的文本</font>

### 5. 链接和图片
#### 1. 链接:
链接的语法分为**行链接**和**引用链接**
| 名称 | 行链接 | 引用链接 |
| :--: | :--: | :--: |
| 语法 | `[文字](URL+标题)` | `[文字][引用ID]` + 后文定义 |
| 可读性 | 较差，链接是在正文中 | 极佳，正文非常干净 |
| 维护性 | 较差，同一链接修改需要一个一个替换 | 佳，只需要替换一次定义 |
| 使用场景 | 一次性，不重复的链接 | 重复使用的链接（如官网，教学网址等）|
1. **行内链接**：直接将链接的所有信息写在行内
    - 基础语法：
    访问[谷歌](https://www.google.com)进行搜索。
    ```markdown
    访问[谷歌](https://www.google.com)进行搜索。
    ```

    - 添加标题（悬停提示）：
    访问[谷歌](https://www.google.com “前往Google首页”)进行搜索
    ```markdown
    访问[谷歌](https://www.google.com “前往Google首页”)进行搜索。
    ```
2. **引用链接**：将连接的名称和地址分开管理，由两部分组成
    - 语法结构
    **第一部分：在正文中使用链接标记**
    这是一个重要的参考资源[1]，更多信息请见[官方网站]。
    **第二部分：在文档的任意位置（通常在文末）定义来链接网址**
    [1]: https://example.com/reference
    [官方网站]: https://example.com “我们的首页”
    ```markdown
        **第一部分：在正文中使用链接标记**
        这是一个重要的参考资源[1]，更多信息请见[官方网站]。
        **第二部分：在文档的任意位置（通常在文末）定义来链接网址**
        [1]: https://example.com/reference
        [官方网站]: https://example.com “我们的首页”
    ```

    - 实际用例:
        欢迎学习Markdown，建议查看[原始语法文档]和[快速入门指南]。
        [原始语法文档]: https://daringfireball.net/projects/markdown/
        [快速入门指南]: https://www.markdownguide.org
    ```markdown
    欢迎学习Markdown，建议查看[原始语法文档]和[快速入门指南]。
    [原始语法文档]: https://daringfireball.net/projects/markdown/
    [快速入门指南]: https://www.markdownguide.org
    ```

#### 2. 图片
1. **基本行内图片**
语法结构：
`![图片替代文本](图片URL)`
- `!`：表示图片，**不是普通链接**
- `[]`：内写**替代文本**（图片加载失败时显示，对无障碍访问至关重要）
- `()`：内写**图片地址**（可以是网络URL[^2]或本地路径）
2. **带标题（悬停提示）的图片**
在URL后添加用**双引号**包裹的标题
`![替代文本](图片URL “鼠标悬停时显示的标题”)`
实例：
`![山峰风景](https://example.com/mountain.jpg “摄于2023年阿尔卑斯山”)`
3. **图片链接**
点击图片条船到链接，结合图片和链接语法:
`[![替代文本](图片URL)](链接URL)`
实例：
`[![GitHub Octocat](https://github.githubassets.com/images/modules/logos_page/Octocat.png)](https://github.com)`
[![GitHub Octocat](https://github.githubassets.com/images/modules/logos_page/Octocat.png)](https://github.com)
4. **引用式图片**
与引用式链接相同，尤其适用于图片连续使用的情况
- **第一部分：在正文标记图片**
  这是我们的logo：![公司Logo][logo]
  ```markdown
  **第一部分：在正文标记图片**
  这是我们的logo：![公司Logo][logo]
  ```
- **第二部分：在文档的任意位置（通常在末尾）定义图片地址**
  [logo]: https://example.com/logo.png “我们的官方Logo”
  ```markdown
  **第二部分：在文档的任意位置（通常在末尾）定义图片地址**
  [logo]: https://example.com/logo.png “我们的官方Logo”
  ```

## 四、进阶部分内容

### 1. 转义符（显示特殊符号）
先前用到的所有符号都不能直接使用，否则会有语义冲突，这是我们可以使用转义符**'\'**来进行表示。
下面是一些例子
   反引号:\`
   星号:\*
   下划线:\_
   大括号:\{\}
   方括号:\[\]
   井号:\#
   加号:\+
   减号:\-
   点号:\.
   感叹号:\!
   管道符:\|

### 2. 表格
markdown中的表格由'\|'来表示
```markdown
   | 表头1 | 表头2 | 表头3 | ...... |
   | ··· | ··· | ··· | ······ |
```
**对齐方式：**
   有三种对齐方式：左对齐，右对齐，居中对齐
   | 左对齐 | 右对齐 | 居中对齐 |
   | :---- | ----: | :---: |
   | a | b | c |
   ```
      | 左对齐 | 右对齐 | 居中对齐 |
      | :---- | ----: | :---: |
      | a | b | c |
   ```
**注意：**
对齐符号由 至少三个 - 组成，前后必须有至少一个管道符 |。
- **长度灵活**：- 的长度可以超过3个（如 -----），且同一列的长度通常会影响最终列宽。
    | 短 | 中等长度 | 非常长的表头 |
    | :- | :------: | ----------: |
    | a  |    b     |           c |
  ```
    | 短 | 中等长度 | 非常长的表头 |
    | :- | :------: | ----------: |
    | a  |    b     |           c |
  ```
- **空格可选**：管道符 `|` 和内容之间的空格只是为了美观，非必须。
    |紧凑|写法|示例|
    |:-|:-:|-:|
    |a|b|c|
    ```
    |紧凑|写法|示例|
    |:-|:-:|-:|
    |a|b|c|
    ```
- 表格对齐行中的 `-` 数量**不需要**与表头的列数相匹配，它只作用于单个列。要求是“**每一列的对齐符号必须独立且完整**”。
  | 姓名         | 年龄 | 城市        | 说明 |
  | :----------- | :--: | ----------- | ---- |
  | 张三         |  25  | 北京        | ...  |    
解读：
- `:-----------` 控制的是“姓名”列（左对齐）
- `:--:` 控制的是“年龄”列（居中对齐）
- `-----------:` 控制的是“城市”列（右对齐）
- `----` 控制的是“说明”列（默认左对齐）
每个控制单元都是独立的，`-` 的数量只决定该列的装饰长度，与其他列无关。

### 6. 锚点（设置目录）
锚点（Anchor）用于实现文档内的超链接跳转，点击后直接定位到目标位置。

#### 一、基础锚点：链接到标题
这是**最常用、最可靠**的锚点用法，因为Markdown标题会自动生成锚点ID。

1. **创建锚点目标（目的地）**
   任何标题都会自动生成一个锚点ID：
2. **创建指向锚点的链接**
   语法结构：
   [跳转到项目背景](#11-项目背景)
   `[跳转到项目背景](#11-项目背景)`
   ==关键点:==
   - 链接中的`#`是锚点标识符
   - ID由标题文本自动生成，规则是
     1. 转为小写
     2. 删除标点符号（`!@$%^&*()+=`等）
     3. 空格替换为连字符`-`
     4. 连续空格/连字符合并成一个
   - 自动生成的ID示例：
     | 原始标题 | 生成的锚点ID | 链接写法 |
     | --- | --- | --- |
     | `# 快速开始` | `快速开始` | `[链接](#快速开始)` |
     | `## Step 1: Install` | `step-1-install` | `[链接](#step-1-install)`
     | `### A.B.测试!!` | `ab测试` | `	[链接](#ab测试)` |
     | `## 配置 - 高级 (选项` | `配置-高级-选项` | `[链接](#配置-高级-选项)` |
#### 二、手动搓锚点（任意位置）
当需要链接到**非标题位置**时，可使用HTML的\<a\>标签来手动创建锚点

**语法**
```
<!-- 1. 在目标位置创建命名锚点 -->
<a id="自定义锚点名称"></a>
这里是你要跳转到的内容...

<!-- 2. 创建指向该锚点的链接 -->
[跳转到自定义位置](#自定义锚点名称)
```

#### 三、跨页面锚点链接
链接到另一个Markdown文件的特定标题：
```
[跳转到另一个文件的章节](../other-file.md#章节标题)
```

[^2]:URL（Uniform Resource Locator，统一资源定位符），俗称 “网址”，是互联网上用于定位和访问资源的标准地址。
