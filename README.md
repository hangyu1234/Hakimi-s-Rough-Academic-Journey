# 笔记食用说明
## 一、为什么会有这个笔记仓库
1. 我们在学习过程中为防止遗忘所做的个人总结的笔记
2. 个人认为这些笔记可能对他人有所帮助
3. 希望给后世留下些东西
## 二、笔记特点
1. 笔记由不同的人编写
2. 笔记统一为Markdown格式编写
3. 笔记内容高度凝练，可能有遗漏
4. 笔记仅代表编者本人见解
5. 笔记可能会存在一些错误，欢迎指正，指正是请注明是哪个笔记（文件路径），哪一行或哪些行
6. 笔记将涵盖各个专业
7. 笔记统一存储在GitHub仓库中，鉴于有些同学打不开该GitHub仓库，可以到阿里云codeup查看该笔记，阿里云codeup需要申请加入该组织，加入该组织后方可看到仓库，其中标注为观察者的为笔记读者，标注为开发者的为笔记编者
8. 相同笔记可以由不同人编写，多个内容相同的笔记可以由编者自行协商是共存还是融合成一个笔记，如果多个笔记选择共存的话请统一放置在一个文件夹中
9. 在每篇笔记里还可能会会有编者在编写笔记中围绕该笔记所具有的一些资源，如作业项目的实现等
## 三、笔记结构
笔记基本按其内容分属不同的文件夹中（可能还会有其他的文件夹）
1. 教材笔记（以出版社为子目录）
2. 国外课程笔记（以国外大学名称为一级子目录，以专业类为二级子目录）
3. 专业书籍笔记（以专业类为一级子目录，以专业类中的领域内容为二级子目录）
4. 其他笔记（以专业类为一级子目录，以专业类中的领域内容为二级子目录）
5. 读书笔记（以文档创作者为一级子目录，以出版社为二级子目录）
6. 笔记工具（以对应笔记编码为一级子目录）
## 四、各类笔记功能介绍
1. 教材笔记：根据每个学校每门课程对应的教材的笔记
2. 国外课程笔记：根据国外一些优质课程内容所编写的笔记
3. 专业书籍笔记：根据对专业有所帮助的一些参考书籍的笔记
4. 其他笔记：根据除以上内容之外的其他与专业相关的内容所编写的笔记
5. 读书笔记：根据编者阅读的课外书籍所编写的笔记
6. 笔记工具：为了使笔记完整呈现所需要的其他文件
## 五、笔记编码
笔记分为本部编码和支部编码
本部编码的笔记由仓库拥有人Empire IV编写
本部编码由三部分组成：编码年份-该笔记在当年笔记的序号-B
（读书笔记最后一位编码为A）
支部编码的笔记由其他编者编写
支部编码由三部分组成：编码年份-该笔记在当年笔记的序号-编者代号

3. 编者上传任何内容请通知本部负责人
